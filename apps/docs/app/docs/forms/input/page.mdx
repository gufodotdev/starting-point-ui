# Input

A Tailwind CSS input component for forms.

```html preview
<div class="space-y-2 w-full max-w-64">
  <label class="label" for="name">Name</label>
  <input class="input" type="text" id="name" placeholder="Enter your name" />
</div>
```

---

## Sizes

Inputs come in three sizes that match button heights for consistent layouts.

```html preview
<div class="space-y-6 w-full max-w-64">
  <div class="space-y-2">
    <label class="label" for="input-sm">Small</label>
    <input class="input input-sm" type="text" id="input-sm" placeholder="Input sm" />
  </div>
  <div class="space-y-2">
    <label class="label" for="input-base">Base</label>
    <input class="input" type="text" id="input-base" placeholder="Input base" />
  </div>
  <div class="space-y-2">
    <label class="label" for="input-lg">Large</label>
    <input class="input input-lg" type="text" id="input-lg" placeholder="Input lg" />
  </div>
</div>
```

## Disabled

Disabled inputs appear faded and cannot be edited.

```html preview
<div class="space-y-2 w-full max-w-64">
  <label class="label" for="disabled-input">Disabled</label>
  <input class="input" type="text" id="disabled-input" placeholder="Disabled input" disabled />
</div>
```

---

## Invalid state

Inputs show a red ring when marked as invalid. Use `aria-describedby` to associate error messages with the input.

```html preview
<div class="space-y-2 w-full max-w-64">
  <label class="label" for="email-invalid">Email address</label>
  <input
    class="input"
    type="email"
    id="email-invalid"
    value="johndoe"
    aria-invalid="true"
    aria-describedby="email-error"
  />
  <p id="email-error" class="text-sm text-destructive">Not a valid email address</p>
</div>
```

## Class reference

| Class | Description |
| --- | --- |
| `input` | Base input styles with focus ring and validation states |
| `input-sm` | Small input, matches btn-sm |
| `input-base` | Base input size (default), matches btn-base |
| `input-lg` | Large input, matches btn-lg |
