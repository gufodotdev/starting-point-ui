# Dialog

A Tailwind CSS dialog component for modals, sheets, and non-modal dialogs.

```html preview
<button class="btn" data-sp-toggle="dialog" data-sp-target="#demo-modal">
  Open Dialog
</button>

<dialog id="demo-modal" class="dialog">
  <div
    class="dialog-backdrop data-[state=open]:animate-in data-[state=closed]:animate-out data-[state=open]:fade-in-0 data-[state=closed]:fade-out-0"
  ></div>
  <div
    class="dialog-panel data-[state=open]:animate-in data-[state=closed]:animate-out data-[state=open]:fade-in-0 data-[state=open]:zoom-in-95 data-[state=closed]:fade-out-0 data-[state=closed]:zoom-out-95 duration-200"
  >
    <button
      class="btn btn-ghost btn-icon absolute top-2 right-2"
      aria-label="Close"
      data-sp-dismiss="dialog"
    >
      <icon-x />
    </button>
    <h2 class="text-lg font-semibold mb-2">Dialog Title</h2>
    <p class="text-muted-foreground mb-4">
      This is a modal dialog. Click the backdrop or press Escape to close.
    </p>
    <div class="flex gap-2 justify-end">
      <button class="btn btn-outline" data-sp-dismiss="dialog">Cancel</button>
      <button class="btn" data-sp-dismiss="dialog">Confirm</button>
    </div>
  </div>
</dialog>
```

### How it works

Use `data-sp-toggle="dialog"` with `data-sp-target` to open a dialog, and `data-sp-dismiss="dialog"` to close it:

- Add `data-sp-toggle="dialog"` and `data-sp-target="#dialog-id"` to any button to open a dialog
- Add `data-sp-dismiss="dialog"` to buttons inside the dialog to close it
- Clicking the backdrop or pressing Escape also closes the dialog (unless using static backdrop)
- The [`open`](https://developer.mozilla.org/en-US/docs/Web/HTML/Element/dialog#open) attribute controls the visibility of the dialog. You can manually add it if you need the dialog to be open on page load.

### JavaScript API

For programmatic control, use the global `StartingPointUI.dialog` module:

```js
const dialog = document.querySelector("#my-dialog");

StartingPointUI.dialog.open(dialog);
StartingPointUI.dialog.close(dialog);
StartingPointUI.dialog.toggle(dialog);
```

## Animation

The dialog sets `data-state="open"` or `data-state="closed"` on the backdrop and panel elements, which you can use to trigger CSS animations.

The examples on this site use [tw-animate-css](https://github.com/Wombosvideo/tw-animate-css) for animation utilities, but you can use any animation library or write your own CSS.

```html preview
<button
  class="btn btn-outline"
  data-sp-toggle="dialog"
  data-sp-target="#anim-none"
>
  None
</button>
<button
  class="btn btn-outline"
  data-sp-toggle="dialog"
  data-sp-target="#anim-zoom"
>
  Fade + Zoom
</button>
<button
  class="btn btn-outline"
  data-sp-toggle="dialog"
  data-sp-target="#anim-slide-bottom"
>
  Slide from Bottom
</button>
<button
  class="btn btn-outline"
  data-sp-toggle="dialog"
  data-sp-target="#anim-slow"
>
  Slow (500ms)
</button>

<dialog id="anim-none" class="dialog">
  <div class="dialog-backdrop"></div>
  <div class="dialog-panel">
    <h2 class="text-lg font-semibold mb-2">No Animation</h2>
    <p class="text-muted-foreground mb-4">
      This is the default behavior. It's up to you to add the animations you
      want.
    </p>
    <button class="btn" data-sp-dismiss="dialog">Close</button>
  </div>
</dialog>

<dialog id="anim-zoom" class="dialog">
  <div
    class="dialog-backdrop data-[state=open]:animate-in data-[state=closed]:animate-out data-[state=open]:fade-in-0 data-[state=closed]:fade-out-0"
  ></div>
  <div
    class="dialog-panel data-[state=open]:animate-in data-[state=closed]:animate-out data-[state=open]:fade-in-0 data-[state=open]:zoom-in-95 data-[state=closed]:fade-out-0 data-[state=closed]:zoom-out-95 duration-200"
  >
    <h2 class="text-lg font-semibold mb-2">Fade + Zoom</h2>
    <p class="text-muted-foreground mb-4">
      The backdrop fades while the panel fades and zooms.
    </p>
    <button class="btn" data-sp-dismiss="dialog">Close</button>
  </div>
</dialog>

<dialog id="anim-slide-bottom" class="dialog">
  <div
    class="dialog-backdrop data-[state=open]:animate-in data-[state=closed]:animate-out data-[state=open]:fade-in-0 data-[state=closed]:fade-out-0 duration-300"
  ></div>
  <div
    class="dialog-panel data-[state=open]:animate-in data-[state=closed]:animate-out data-[state=open]:fade-in-0 data-[state=open]:slide-in-from-bottom data-[state=closed]:fade-out-0 data-[state=closed]:slide-out-to-bottom duration-300"

    <h2 class="text-lg font-semibold mb-2">Slide from Bottom</h2>
    <p class="text-muted-foreground mb-4">

    </p>
    <button class="btn" data-sp-dismiss="dialog">Close</button>
  </div>
</dialog>

<dialog id="anim-slow" class="dialog">
  <div
    class="dialog-backdrop data-[state=open]:animate-in data-[state=closed]:animate-out data-[state=open]:fade-in-0 data-[state=closed]:fade-out-0 duration-500"
  ></div>
  <div
    class="dialog-panel data-[state=open]:animate-in data-[state=closed]:animate-out data-[state=open]:fade-in-0 data-[state=open]:zoom-in-95 data-[state=closed]:fade-out-0 data-[state=closed]:zoom-out-95 duration-500"

    <h2 class="text-lg font-semibold mb-2">Slow Animation</h2>
    <p class="text-muted-foreground mb-4">

    </p>
    <button class="btn" data-sp-dismiss="dialog">Close</button>
  </div>
</dialog>
```

### How it works

Use `data-[state=open]` and `data-[state=closed]` selectors to apply different animations for entering and exiting:

- `data-[state=open]:animate-in` — triggers the enter animation
- `data-[state=closed]:animate-out` — triggers the exit animation

See the [tw-animate-css documentation](https://github.com/Wombosvideo/tw-animate-css) for all available animation utilities.

## Modal vs Non-modal

This component uses `dialog.show()` instead of `dialog.showModal()`, so the backdrop is what creates the modal behavior. Add `.dialog-backdrop` to block interaction with the page behind it, or omit it for a non-modal dialog.

```html preview
<button
  class="btn btn-outline"
  data-sp-toggle="dialog"
  data-sp-target="#modal-example"
>
  Open Modal
</button>
<button
  class="btn btn-outline"
  data-sp-toggle="dialog"
  data-sp-target="#nonmodal-example"
>
  Open Non-modal
</button>

<dialog id="modal-example" class="dialog">
  <div
    class="dialog-backdrop data-[state=open]:animate-in data-[state=closed]:animate-out data-[state=open]:fade-in-0 data-[state=closed]:fade-out-0"
  ></div>
  <div
    class="dialog-panel data-[state=open]:animate-in data-[state=closed]:animate-out data-[state=open]:fade-in-0 data-[state=open]:zoom-in-95 data-[state=closed]:fade-out-0 data-[state=closed]:zoom-out-95 duration-200"
  >
    <h2 class="text-lg font-semibold mb-2">Modal</h2>
    <p class="text-muted-foreground mb-4">
      Background is blocked. Click backdrop or press Escape to close.
    </p>
    <button class="btn" data-sp-dismiss="dialog">Close</button>
  </div>
</dialog>

<dialog id="nonmodal-example" class="dialog">
  <div
    class="dialog-panel data-[state=open]:animate-in data-[state=closed]:animate-out data-[state=open]:fade-in-0 data-[state=open]:zoom-in-95 data-[state=closed]:fade-out-0 data-[state=closed]:zoom-out-95 duration-200"
  >
    <h2 class="text-lg font-semibold mb-2">Non-modal</h2>
    <p class="text-muted-foreground mb-4">
      You can interact with the page behind this dialog.
    </p>
    <button class="btn" data-sp-dismiss="dialog">Close</button>
  </div>
</dialog>
```

## Static backdrop

Use `data-sp-backdrop="static"` to prevent closing when clicking the backdrop or pressing Escape.

```html preview
<button
  class="btn btn-destructive"
  data-sp-toggle="dialog"
  data-sp-target="#static-modal"
>
  Delete
</button>

<dialog id="static-modal" class="dialog" data-sp-backdrop="static">
  <div
    class="dialog-backdrop data-[state=open]:animate-in data-[state=closed]:animate-out data-[state=open]:fade-in-0 data-[state=closed]:fade-out-0"
  ></div>
  <div
    class="dialog-panel data-[state=open]:animate-in data-[state=closed]:animate-out data-[state=open]:fade-in-0 data-[state=open]:zoom-in-95 data-[state=closed]:fade-out-0 data-[state=closed]:zoom-out-95 duration-200"
  >
    <h2 class="text-lg font-semibold mb-2">Are you sure?</h2>
    <p class="text-muted-foreground mb-4">
      This action cannot be undone. Backdrop clicks and Escape are disabled.
    </p>
    <div class="flex gap-2 justify-end">
      <button class="btn btn-outline" data-sp-dismiss="dialog">Cancel</button>
      <button class="btn btn-destructive" data-sp-dismiss="dialog">
        Delete
      </button>
    </div>
  </div>
</dialog>
```

## Accessibility

This component uses the native `<dialog>` element, which provides built-in accessibility features like focus trapping, Escape key handling, and proper screen reader announcements.

```html preview
<button
  class="btn btn-outline"
  data-sp-toggle="dialog"
  data-sp-target="#accessible-dialog"
>
  Open Accessible Dialog
</button>

<dialog
  id="accessible-dialog"
  class="dialog"
  aria-labelledby="dialog-title"
  aria-describedby="dialog-description"
>
  <div
    class="dialog-backdrop data-[state=open]:animate-in data-[state=closed]:animate-out data-[state=open]:fade-in-0 data-[state=closed]:fade-out-0"
  ></div>
  <div
    class="dialog-panel data-[state=open]:animate-in data-[state=closed]:animate-out data-[state=open]:fade-in-0 data-[state=open]:zoom-in-95 data-[state=closed]:fade-out-0 data-[state=closed]:zoom-out-95 duration-200"
  >
    <button
      class="btn btn-ghost btn-icon absolute top-2 right-2"
      aria-label="Close dialog"
      data-sp-dismiss="dialog"
    >
      <icon-x />
    </button>
    <h2 id="dialog-title" class="text-lg font-semibold mb-2">
      Accessible Dialog
    </h2>
    <p id="dialog-description" class="text-muted-foreground mb-4">
      This dialog uses ARIA attributes to improve screen reader support.
    </p>
    <button class="btn w-full" data-sp-dismiss="dialog">Close</button>
  </div>
</dialog>
```

### Additional recommended attributes

| Attribute                    | Element  | Description                                 |
| ---------------------------- | -------- | ------------------------------------------- |
| `aria-labelledby="title-id"` | `dialog` | References the dialog's title element       |
| `aria-describedby="desc-id"` | `dialog` | References the dialog's description         |
| `aria-label="Close"`         | `button` | Labels icon-only buttons for screen readers |

## Class reference

All available classes for the dialog component.

| Class             | Description                                             |
| ----------------- | ------------------------------------------------------- |
| `dialog`          | Base dialog container                                   |
| `dialog-backdrop` | Overlay that blocks interaction with the page behind it |
| `dialog-panel`    | The dialog content panel                                |

---

## Data attributes

| Attribute                   | Element                                | Description                                    |
| --------------------------- | -------------------------------------- | ---------------------------------------------- |
| `data-sp-toggle="dialog"`   | `button`                               | Opens the dialog specified by `data-sp-target` |
| `data-sp-target="#id"`      | `button`                               | The ID of the dialog to open                   |
| `data-sp-dismiss="dialog"`  | `button`                               | Closes the parent dialog                       |
| `data-sp-backdrop="static"` | `dialog`                               | Prevents closing on backdrop click or Escape   |
| `data-state="open"`         | `.dialog-backdrop`<br/>`.dialog-panel` | Set when the dialog is open                    |
| `data-state="closed"`       | `.dialog-backdrop`<br/>`.dialog-panel` | Set when the dialog is closing                 |
