A Tailwind CSS input component for forms.

```html preview
<div class="space-y-2 w-full max-w-64">
  <label class="label" for="name">Name</label>
  <input class="input" type="text" id="name" placeholder="Enter your name" />
</div>
```

## Disabled

Disabled inputs appear faded and cannot be edited.

```html preview
<div class="space-y-2 w-full max-w-64">
  <label class="label" for="disabled-input">Disabled</label>
  <input
    class="input"
    type="text"
    id="disabled-input"
    placeholder="Disabled input"
    disabled
  />
</div>
```

## Invalid state

Inputs show a red ring when marked as invalid. Use `aria-describedby` to associate error messages with the input.

```html preview
<div class="space-y-2 w-full max-w-64">
  <label class="label" for="email-invalid">Email address</label>
  <input
    class="input"
    type="email"
    id="email-invalid"
    value="johndoe"
    aria-invalid="true"
    aria-describedby="email-error"
  />
  <p id="email-error" class="text-sm text-destructive">
    Not a valid email address
  </p>
</div>
```

## How it works

The input component is a CSS-only styled native input element. It supports various input types, size variants, and disabled and invalid states.

### Structure

An input is a native `<input>` element with the `.input` class:

```html
<input class="input" type="text" placeholder="Enter text" />
```

### Accessibility

For proper screen reader support, pair inputs with labels using matching `id` and `for` attributes:

```html
<div class="space-y-2">
  <label class="label" for="email">Email</label>
  <input class="input" type="email" id="email" placeholder="you@example.com" />
</div>
```

Additional accessibility attributes:

| Attribute                    | Description                                                      |
| ---------------------------- | ---------------------------------------------------------------- |
| `id`                         | Add to `.input` to associate with a label                        |
| `aria-invalid="true"`        | Add to `.input` to indicate the input has an error               |
| `aria-describedby="id"`      | Add to `.input` to associate with an error message or hint       |
| `disabled`                   | Add to `.input` to prevent interaction                           |
| `required`                   | Add to `.input` to indicate the field is required                |

```html
<div class="space-y-2">
  <label class="label" for="email">Email</label>
  <input
    class="input"
    type="email"
    id="email"
    aria-invalid="true"
    aria-describedby="email-error"
    required
  />
  <p id="email-error" class="text-sm text-destructive">
    Please enter a valid email address
  </p>
</div>
```

### Class reference

All available classes for the input component.

| Class        | Description                                                    |
| ------------ | -------------------------------------------------------------- |
| `input`      | Add to `<input>` element for styled input with focus ring      |
| `input-sm`   | Add to `.input` for small size (matches btn-sm)                |
| `input-base` | Add to `.input` for base size (default, matches btn-base)      |
| `input-lg`   | Add to `.input` for large size (matches btn-lg)                |

```html
<input class="input" type="text" />
<input class="input input-sm" type="text" />
<input class="input input-lg" type="text" />
```
