A Tailwind CSS tabs component for organizing content into switchable panels.

```html preview
<div class="w-80">
  <div class="tab-list" role="tablist" aria-label="Example tabs">
    <button
      type="button"
      class="tab active"
      role="tab"
      aria-selected="true"
      data-sp-toggle="tab"
      data-sp-target="#panel-1"
      tabindex="0"
    >
      Account
    </button>
    <button
      type="button"
      class="tab"
      role="tab"
      aria-selected="false"
      data-sp-toggle="tab"
      data-sp-target="#panel-2"
      tabindex="-1"
    >
      Password
    </button>
    <button
      type="button"
      class="tab"
      role="tab"
      aria-selected="false"
      data-sp-toggle="tab"
      data-sp-target="#panel-3"
      tabindex="-1"
    >
      Settings
    </button>
  </div>
  <div id="panel-1" class="tab-panel active pt-4" role="tabpanel">
    <p>Manage your account settings and preferences.</p>
  </div>
  <div id="panel-2" class="tab-panel pt-4" role="tabpanel">
    <p>Change your password and security options.</p>
  </div>
  <div id="panel-3" class="tab-panel pt-4" role="tabpanel">
    <p>Configure application settings.</p>
  </div>
</div>
```

### How it works

Use `data-sp-toggle="tab"` with `data-sp-target` to connect tabs to their panels:

- Add `data-sp-toggle="tab"` and `data-sp-target="#panel-id"` to each tab button
- Add the `.active` class to the initially selected tab and its panel
- Clicking a tab activates it and shows the corresponding panel

### JavaScript API

For programmatic control, use the global `StartingPointUI.tabs` module:

```js
const tab = document.querySelector("#my-tab");

StartingPointUI.tabs.select(tab);
```

## Keyboard navigation

Tabs support full keyboard navigation following the WAI-ARIA tabs pattern:

| Key          | Action                   |
| ------------ | ------------------------ |
| `ArrowLeft`  | Move to the previous tab |
| `ArrowRight` | Move to the next tab     |
| `Home`       | Move to the first tab    |
| `End`        | Move to the last tab     |

Navigation wraps around - pressing `ArrowRight` on the last tab moves to the first tab, and vice versa.

## Accessibility

For accessible tabs, add the proper ARIA attributes:

```html preview
<div class="w-80">
  <div class="tab-list" role="tablist" aria-label="Settings tabs">
    <button
      type="button"
      class="tab active"
      role="tab"
      id="tab-general"
      aria-selected="true"
      aria-controls="panel-general"
      data-sp-toggle="tab"
      data-sp-target="#panel-general"
      tabindex="0"
    >
      General
    </button>
    <button
      type="button"
      class="tab"
      role="tab"
      id="tab-notifications"
      aria-selected="false"
      aria-controls="panel-notifications"
      data-sp-toggle="tab"
      data-sp-target="#panel-notifications"
      tabindex="-1"
    >
      Notifications
    </button>
  </div>
  <div
    id="panel-general"
    class="tab-panel active pt-4"
    role="tabpanel"
    aria-labelledby="tab-general"
  >
    <p>General settings content.</p>
  </div>
  <div
    id="panel-notifications"
    class="tab-panel pt-4"
    role="tabpanel"
    aria-labelledby="tab-notifications"
  >
    <p>Notification preferences content.</p>
  </div>
</div>
```

### Recommended attributes

| Attribute                     | Element      | Description                                 |
| ----------------------------- | ------------ | ------------------------------------------- |
| `role="tablist"`              | `.tab-list`  | Identifies the container as a tablist       |
| `role="tab"`                  | `.tab`       | Identifies the button as a tab              |
| `role="tabpanel"`             | `.tab-panel` | Identifies the content as a tabpanel        |
| `aria-label`                  | `.tab-list`  | Provides an accessible name for the tablist |
| `aria-selected="true\|false"` | `.tab`       | Indicates the selected state                |
| `aria-controls="panel-id"`    | `.tab`       | References the controlled panel             |
| `aria-labelledby="tab-id"`    | `.tab-panel` | References the tab that controls this panel |
| `tabindex="0\|-1"`            | `.tab`       | Only the active tab should be in tab order  |

## Styling panels

Avoid applying `display` properties directly to `.tab-panel` since the `.active` class controls visibility with `display: block`. If you need flex or grid layouts, use a wrapper inside the panel:

```html
<!-- Don't do this -->
<div class="tab-panel active flex gap-4">...</div>

<!-- Do this instead -->
<div class="tab-panel active">
  <div class="flex gap-4">...</div>
</div>
```

## Class reference

All available classes for the tabs component.

| Class              | Description                         |
| ------------------ | ----------------------------------- |
| `tab-list`         | Container for tab buttons           |
| `tab`              | Individual tab button               |
| `tab.active`       | Active state for the selected tab   |
| `tab-panel`        | Content panel associated with a tab |
| `tab-panel.active` | Active state showing the panel      |

## Data attributes

| Attribute              | Element | Description                                         |
| ---------------------- | ------- | --------------------------------------------------- |
| `data-sp-toggle="tab"` | `.tab`  | Enables tab switching on click                      |
| `data-sp-target="#id"` | `.tab`  | The ID of the panel to show when this tab is active |
