A Tailwind CSS textarea component for forms.

```html preview
<div class="space-y-2 w-full max-w-64">
  <label class="label" for="message">Message</label>
  <textarea
    class="textarea"
    id="message"
    placeholder="Enter your message..."
  ></textarea>
</div>
```

---

## Auto-sizing

Textareas automatically grow based on content using `field-sizing: content`.

```html preview
<textarea
  class="textarea w-full max-w-64"
  placeholder="Type something and watch it grow..."
></textarea>
```

---

## Disabled

Disabled textareas appear faded and cannot be edited.

```html preview
<textarea
  class="textarea w-full max-w-64"
  placeholder="Disabled textarea"
  disabled
></textarea>
```

---

## Invalid state

Textareas show a red ring when marked as invalid.

```html preview
<div class="space-y-2 w-full max-w-64">
  <label class="label" for="bio">Bio</label>
  <textarea
    class="textarea"
    id="bio"
    aria-invalid="true"
    aria-describedby="bio-error"
  >
This bio is too long...</textarea
  >
  <p id="bio-error" class="text-sm text-destructive">
    Bio must be under 200 characters
  </p>
</div>
```

## Customization

Combine textarea classes with Tailwind utilities for custom styling.

```html preview
<div class="space-y-6 w-full max-w-80">
  <textarea
    class="textarea min-h-32"
    placeholder="Fixed minimum height..."
  ></textarea>
  <textarea
    class="textarea resize-none"
    placeholder="Resize disabled..."
  ></textarea>
  <div class="relative">
    <textarea
      class="textarea pb-10"
      placeholder="Write a message..."
    ></textarea>
    <button class="btn btn-sm absolute bottom-4 right-2">Send</button>
  </div>
</div>
```

## How it works

The textarea component is a CSS-only styled native textarea element. It includes auto-sizing based on content and supports disabled and invalid states.

### Structure

A textarea is a native `<textarea>` element with the `.textarea` class:

```html
<textarea class="textarea" placeholder="Enter text..."></textarea>
```

### Accessibility

For proper screen reader support, pair textareas with labels using matching `id` and `for` attributes:

```html
<label class="label" for="message">Message</label>
<textarea class="textarea" id="message" placeholder="Enter your message..."></textarea>
```

Additional accessibility attributes:

| Attribute                    | Description                                                        |
| ---------------------------- | ------------------------------------------------------------------ |
| `id`                         | Add to `.textarea` to associate with a label                       |
| `aria-invalid="true"`        | Add to `.textarea` to indicate the textarea has an error           |
| `aria-describedby="id"`      | Add to `.textarea` to associate with an error message or hint      |
| `disabled`                   | Add to `.textarea` to prevent interaction                          |
| `required`                   | Add to `.textarea` to indicate the field is required               |

### Class reference

All available classes for the textarea component.

| Class      | Description                                                         |
| ---------- | ------------------------------------------------------------------- |
| `textarea` | Add to `<textarea>` element for styled textarea with auto-sizing    |

```html
<textarea class="textarea" placeholder="Enter text..."></textarea>
```
