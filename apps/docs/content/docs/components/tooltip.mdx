A Tailwind CSS tooltip component for displaying informational text on hover or when the element receives keyboard focus.

```html preview
<div class="tooltip">
  <button class="btn" data-sp-toggle="tooltip" aria-describedby="tip-1">Hover me</button>
  <div class="tooltip-content" id="tip-1" role="tooltip">Tooltip text</div>
</div>
```

---

## Placement

Use `data-sp-placement` to control the tooltip position (default is `top`).

```html preview
<div class="tooltip" data-sp-placement="left">
  <button class="btn" data-sp-toggle="tooltip" aria-describedby="tip-left">Left</button>
  <div class="tooltip-content" id="tip-left" role="tooltip">Left tooltip</div>
</div>
<div class="tooltip" data-sp-placement="top">
  <button class="btn" data-sp-toggle="tooltip" aria-describedby="tip-top">Top</button>
  <div class="tooltip-content" id="tip-top" role="tooltip">Top tooltip</div>
</div>
<div class="tooltip" data-sp-placement="bottom">
  <button class="btn" data-sp-toggle="tooltip" aria-describedby="tip-bottom">Bottom</button>
  <div class="tooltip-content" id="tip-bottom" role="tooltip">Bottom tooltip</div>
</div>
<div class="tooltip" data-sp-placement="right">
  <button class="btn" data-sp-toggle="tooltip" aria-describedby="tip-right">Right</button>
  <div class="tooltip-content" id="tip-right" role="tooltip">Right tooltip</div>
</div>
```

## Animation

The tooltip includes a default fade and zoom animation. Add `no-animation` to the 
content element to disable it, or use your own classes with `data-[state=open]:` selectors.

```html preview
<div class="tooltip">
  <button class="btn" data-sp-toggle="tooltip" aria-describedby="tip-default">Default</button>
  <div class="tooltip-content" id="tip-default" role="tooltip">Default animation</div>
</div>
<div class="tooltip">
  <button class="btn" data-sp-toggle="tooltip" aria-describedby="tip-noanim">No Animation</button>
  <div class="tooltip-content no-animation" id="tip-noanim" role="tooltip">No animation</div>
</div>
<div class="tooltip">
  <button class="btn" data-sp-toggle="tooltip" aria-describedby="tip-slide">
    Slide from Bottom
  </button>
  <div
    class="tooltip-content no-animation data-[state=open]:animate-in data-[state=open]:fade-in-0 data-[state=open]:slide-in-from-bottom"
    id="tip-slide"
    role="tooltip"
  >
    Custom animation
  </div>
</div>
```

The default animations use [tw-animate-css](https://github.com/Wombosvideo/tw-animate-css) utilities. 
You can customize or replace these with your own animations.

## How it works

The tooltip component uses a small JavaScript module that handles showing, hiding, and positioning the tooltip content.

### Structure

A tooltip consists of three parts:

1. **`.tooltip`** - Container that wraps the trigger and content
2. **`[data-sp-toggle="tooltip"]`** - Element that shows the tooltip on hover/focus
3. **`.tooltip-content`** - The floating content panel

```html
<div class="tooltip">
  <button data-sp-toggle="tooltip" aria-describedby="my-tip">Hover me</button>
  <div class="tooltip-content" id="my-tip" role="tooltip">Tooltip text</div>
</div>
```

The arrow element (`.tooltip-arrow`) is automatically created by the JavaScript when the tooltip opens.

### Opening and closing

The tooltip opens automatically when the user hovers over or focuses the trigger element. It closes when 
the mouse leaves or the trigger loses focus. Pressing Escape also closes the tooltip.

### Animation

The tooltip includes a default fade and zoom animation. When shown, the JavaScript sets `data-state="open"` on the content element.

To customize animations, add `no-animation` to disable the defaults and use your own classes with `data-[state=open]:` selectors:

```html
<div
  class="tooltip-content no-animation data-[state=open]:animate-in data-[state=open]:slide-in-from-bottom"
  role="tooltip"
>
  <!-- Custom slide animation -->
</div>
```

### Accessibility

The tooltip JavaScript module shows content on both hover and keyboard focus.

Add `role="tooltip"` to the content element:

```html
<div class="tooltip-content" role="tooltip">Tooltip text</div>
```

For screen reader users, link the trigger to the content with `aria-describedby`.
Add an `id` to the content and point the trigger at it:

```html
<div class="tooltip">
  <button
    class="btn"
    data-sp-toggle="tooltip"
    aria-describedby="save-tip"
  >
    Save
  </button>
  <div class="tooltip-content" id="save-tip" role="tooltip">
    Saves your changes
  </div>
</div>
```

Without `aria-describedby`, the tooltip still works visually but its content
won't be announced by screen readers.

#### Keyboard navigation

| Key      | Action                        |
| -------- | ----------------------------- |
| `Tab`    | Focus trigger to show tooltip |
| `Escape` | Closes the tooltip            |

### Class reference

All available classes for the tooltip component.

| Class             | Description                                       |
| ----------------- | ------------------------------------------------- |
| `tooltip`         | Container that wraps the trigger and content      |
| `tooltip-content` | The floating content that appears on hover/focus  |
| `tooltip-arrow`   | Arrow element (auto-injected by JavaScript)       |
| `no-animation`    | Disables default animation on tooltip-content     |

```html
<div class="tooltip">
  <button data-sp-toggle="tooltip" aria-describedby="my-tip">Hover me</button>
  <div class="tooltip-content" id="my-tip" role="tooltip">Tooltip text</div>
</div>
```

### Data attributes

All data attributes for the tooltip component.

| Attribute                  | Element            | Description                           |
| -------------------------- | ------------------ | ------------------------------------- |
| `data-sp-toggle="tooltip"` | Button/trigger     | Designates the tooltip trigger        |
| `data-sp-placement`        | `.tooltip`         | Content position (default: `top`)     |
| `data-state`               | `.tooltip-content` | Set to `open` when content is visible |

```html
<div class="tooltip" data-sp-placement="bottom">
  <button data-sp-toggle="tooltip" aria-describedby="my-tip">Hover me</button>
  <div class="tooltip-content" id="my-tip" role="tooltip">Tooltip text</div>
</div>
```
